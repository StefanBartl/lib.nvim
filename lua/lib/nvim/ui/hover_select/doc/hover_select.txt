hover-select                                         *hover-select*

A small, minimal helper module for displaying and selecting items in a
floating window relative to the cursor position.

==============================================================================
INTRODUCTION                                                        *hover-select-intro*

hover-select provides a lightweight selection UI intended for plugin authors
and advanced users who need full control over buffers and floating windows.
It is designed as a building block rather than a full-featured picker.

==============================================================================
FEATURES                                                           *hover-select-features*

* Floating window relative to cursor, window, or editor
* Simple list-based selection (one item per line)
* Callback executed when an item is selected
* Vertical-only navigation (horizontal movement disabled)
* Automatic window sizing with minimum and maximum bounds
* Automatic cleanup of buffers and windows
* Dedicated highlight group for the active line

==============================================================================
MULTI-SELECTION                                    *hover-select-multiselect*

hover-select supports multi-line selection via keyboard shortcuts. Users can
mark multiple items before confirming the selection.

Enable multi-selection by passing `multi_select = true` in the options table:
>
    hover_select.open({
      items = { "A", "B", "C" },
      multi_select = true,
      on_select = function(selected, indices)
        -- selected: array of items
        -- indices: array of line numbers (1-based)
      end
    })
<

------------------------------------------------------------------------------
MULTI-SELECT KEYS                              *hover-select-multiselect-keys*

<Tab>           Toggle selection for current line (stays on same line)
<S-Tab>         Toggle selection and move cursor up
<CR>            Confirm selection (all marked items, or current if none)
<Esc> / q       Close without selection

Visual feedback:
- Current line: highlighted with HoverSelectCursor
- Selected lines: highlighted with HoverSelectSelected

------------------------------------------------------------------------------
CALLBACK SIGNATURE                         *hover-select-multiselect-callback*

Single-select mode (default):
>
    on_select = function(selected, index)
      -- selected: string
      -- index: integer (1-based)
    end
<

Multi-select mode (`multi_select = true`):
>
    on_select = function(selected, indices)
      -- selected: string[] (array)
      -- indices: integer[] (array)
    end
<

If no items are marked with <Tab>, pressing <CR> returns the current line
as a single-element array.

==============================================================================
TESTING                                                *hover-select-testing*

Test the multi-selection feature:
>
    :lua require("lib.nvim.ui.hover_select.test_multiselect").test_multi_select()
<

Available test functions:
- test_single_select()        Single-select mode
- test_multi_select()          Basic multi-select
- test_multi_select_long()     Long list (20 items)
- test_file_list()             Realistic file selection

==============================================================================
MODULE STRUCTURE                                                   *hover-select-modules*

lib.nvim.ui.hover_select.buffer
Buffer creation, content updates, and buffer-local options.

lib.nvim.ui.hover_select.window
Floating window creation, size calculation, and lifecycle management.

lib.nvim.ui.hover_select.navigation
Keymaps for navigation, selection, and closing the window.

lib.nvim.ui.hover_select.highlight
Highlight setup and management for the active cursor line.

lib.nvim.ui.hover_select.config
Default configuration for buffers, windows, and layout constraints.

lib.nvim.ui.hover_select.@types
EmmyLua type definitions for options and internal state.

==============================================================================
INTERACTION                                                        *hover-select-keys*

Normal mode mappings inside the hover-select buffer:

<CR>            Select the current item
<2-LeftMouse>   Select the clicked item <Esc>           Close the window
q               Close the window

Vertical movement uses the default Neovim keys (j/k, arrow keys).
Horizontal movement is intentionally disabled.

==============================================================================
CONFIGURATION                                                      *hover-select-config*

hover-select ships with sensible defaults for buffer options, window options,
layout, and size constraints. All values can be overridden by passing custom
tables, which are merged with the defaults.

==============================================================================
INTENDED USE                                                       *hover-select-use*

hover-select is intended for:

* custom Neovim plugins
* internal developer tools
* small context-aware selection menus
* situations where minimal UI and maximum control are required
